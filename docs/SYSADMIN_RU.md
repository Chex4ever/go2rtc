# Видеостена — руководство для администратора (русский)

Краткая инструкция по развёртыванию **go2rtc** с **кастомной видеостеной** (пользователи, раскладки, два потока на камеру).  
Полное описание возможностей go2rtc — в основном [README.md](../README.md).

---

## Что это

| Компонент | Назначение |
|-----------|------------|
| **go2rtc** | Подключение к камерам (RTSP, ONVIF, …), выдача видео в браузер |
| **Видеостена** (`/viewer/`) | Оператор входит по логину, выбирает раскладку, смотрит камеры |
| **Админка viewer** (`/viewer/admin.html`) | Вы настраиваете пользователей, раскладки, списки камер, preview |

Ориентир: **отдельный сервер в LAN**, до **~5 пользователей**, **5 раскладок**, **~25 камер**.

---

## Схема

```text
  Операторы (браузер, LAN)
         │
         │  http://SERVER:1984/viewer/
         ▼
  ┌──────────────────────────────────────┐
  │  go2rtc.exe                          │
  │  ┌────────────┐  ┌─────────────────┐ │
  │  │ streams    │  │ модуль viewer   │ │
  │  │ (камеры)   │  │ users, layouts  │ │
  │  └─────┬──────┘  └────────┬────────┘ │
  │        │                  │          │
  │        └────────┬─────────┘          │
  │                 ▼                    │
  │         WebRTC / MSE в браузер       │
  └──────────────────┬───────────────────┘
                     │ RTSP / ONVIF / …
                     ▼
               IP-камеры
```

**Два файла конфигурации:**

| Файл | Кто правит | Содержимое |
|------|------------|------------|
| `go2rtc.yaml` | Администратор | URL камер (`streams:`), порт API, ffmpeg, **блок viewer** |
| `viewer.yaml` | Администратор (веб-админка или редактор) | Пользователи viewer, раскладки, доступ к камерам |

---

## Два потока (сетка и полный экран)

На стене из многих камер экономится трафик: для каждой камеры в раскладке задаются **два имени потока в go2rtc**:

| Режим | Какой поток | Качество |
|-------|-------------|----------|
| **Сетка** (много плиток) | **Preview** (субпоток) | Ниже разрешение / битрейт |
| **Полный экран** (одна камера) | **Основной** поток | Полное качество |

```text
  go2rtc.yaml                    viewer.yaml (раскладка)
  ─────────────                  ───────────────────────
  cam1      ── основной RTSP ──► cameras: [cam1, …]
  cam1_sub  ── субпоток RTSP  ──► preview:
                                    cam1: cam1_sub
```

На стене оператор видит имя **cam1**; интерфейс сам выбирает поток для сетки или fullscreen.

**Пример в `go2rtc.yaml`:**

```yaml
streams:
  cam1: rtsp://admin:pass@192.168.1.101/stream1
  cam1_sub: rtsp://admin:pass@192.168.1.101/stream2
  cam2: rtsp://admin:pass@192.168.1.102/h264
  cam2_sub: rtsp://admin:pass@192.168.1.102/h264?subtype=1
```

Если `preview` не задан, везде используется один и тот же поток.

## Раскладки и сетки

**Раскладка** = один пресет видеостены:

| Сетка | Плиток | Типичное применение |
|-------|--------|---------------------|
| **6** | 3×2 | Небольшая комната, ресепшен |
| **7** | 4×2 (асимметрия) | Смешанные размеры |
| **25** | 5×5 | Диспетчерская |
| **36** | 6×6 | Большая стена (проверьте канал LAN) |

Правила:

- `cameras:` — **разрешённый список**; на раскладку нельзя вывести камеру вне списка.
- Камер не больше, чем плиток в сетке (например, не более 25 для grid 25).
- **Порядок плиток** сохраняется для каждого пользователя в `viewer.yaml` (`user_layout_state`); на ПК можно перетаскивать плитки.

## Авторизация (два уровня)

```text
  Уровень 1 — API go2rtc (go2rtc.yaml → api.username / api.password)
              Защита: config.html, сырой /api/* (кроме обхода для viewer)

  Уровень 2 — Viewer (viewer.yaml + viewer.admin_password)
              Защита: вход /viewer/, раскладки по пользователям
              Админка: /viewer/admin.html (пароль admin_password)
```

| URL | Вход |
|-----|------|
| `http://SERVER:1984/` | Пользователь API go2rtc (если задан) |
| `http://SERVER:1984/viewer/` | **Пользователь viewer** (из `viewer.yaml`) |
| `http://SERVER:1984/viewer/admin.html` | **Пароль viewer admin** (из `go2rtc.yaml`) |

**Запомнить по IP:** при входе можно включить «Remember this device» — с того же IP в LAN пароль не спрашивается до истечения `trust_ip_ttl`. Удобно для стационарных ПК; это не строгая защита.

Снимки доступны только для камер из раскладок данного пользователя.

## Быстрый старт

### 1. Файлы на сервере

```text
C:\go2rtc\
  go2rtc.exe
  go2rtc.yaml
  viewer.yaml          ← создаётся при первом запуске, если нет
```

Пример: [viewer.yaml.example](viewer.yaml.example).

### 2. Включить viewer в `go2rtc.yaml`

```yaml
api:
  listen: ":1984"
  # username: admin
  # password: secret

viewer:
  config: viewer.yaml
  admin_password: changeme    # обязательно для админки viewer
  session_ttl: 24h
  trust_ip_ttl: 720h          # срок «запомнить IP»
  cookie_secure: false        # true только за HTTPS
```

### 3. Добавить потоки камер

```yaml
streams:
  cam1: rtsp://user:pass@192.168.1.101/stream1
  cam1_sub: rtsp://user:pass@192.168.1.101/stream2
```

Проверка: `http://SERVER:1984/` → список streams.

### 4. Пользователи и раскладки

Откройте **`http://SERVER:1984/viewer/admin.html`**, введите `admin_password`.

- **Users** — логин, пароль, список ID раскладок  
- **Layouts** — размер сетки (6 / 7 / 25 / 36), галочки камер из go2rtc  
- **Preview streams** — связка основной → субпоток для режима сетки  

Можно править `viewer.yaml` вручную и перезапустить go2rtc.

### 5. Выдать операторам

URL: **`http://SERVER:1984/viewer/`**

Вход → выбор раскладки → стена. Двойной клик или ⛶ — полный экран (основной поток).

## Сценарии

### Диспетчерская (25 камер)

- Раскладка `wall_25`, сетка **25**, все камеры в allow-list.  
- Для каждой камеры в preview указать **субпоток**.  
- Стационарный ПК в LAN с «Запомнить устройство».  
- Fullscreen + снимок (📷) при инциденте.

### Ресепшен (6 камер)

- Раскладка `lobby`, сетка **6**.  
- Пользователи: `reception` (только lobby), `security` (lobby + parking).

### Планшет / обход

- Тот же URL; на планшете — список или 2 колонки.  
- На touch нет drag; fullscreen — кнопка ⛶.

### Смена порядка камер без админа

- Оператор переставляет плитки на ПК; порядок пишется в `viewer.yaml`.  
- Админ поддерживает только имена потоков и списки камер.

## Возможности оператора (кратко)

| Функция | Примечание |
|---------|------------|
| Звук | По умолчанию выключен; включение по плитке |
| Масштаб / сдвиг | На плитке; pinch на телефоне |
| Снимок | JPEG; запасной вариант — API go2rtc |
| Запись | WebM из браузера (лучше Chrome + WebRTC) |
| Перезапуск | Сохранение в веб-config перезапускает процесс (Windows учтён) |

---

## Чеклист развёртывания

- [ ] Сервер в VLAN камер или LAN; с WAN закрыт порт `:1984`, если не нужен снаружи  
- [ ] Сменены пароли `admin_password` и пользователей viewer  
- [ ] Субпотоки (`camX_sub`) проверены до добавления в preview  
- [ ] Канал: N preview ≈ N × битрейт субпотока (для стены 25)  
- [ ] После обновления **exe** — перезапуск службы (UI вшит в бинарник)  
- [ ] Бэкап `go2rtc.yaml` + `viewer.yaml` вместе  

---

## Устранение неполадок

| Симптом | Что проверить |
|---------|----------------|
| Не входит в viewer | Пользователь в `viewer.yaml`, пароль, ID раскладок |
| Пустой список раскладок | В `layouts:` пользователя те же ключи, что в `viewer.yaml` |
| Чёрная плитка | Поток есть в go2rtc; имя в allow-list раскладки |
| Сетка OK, fullscreen плохо | URL **основного** потока в `go2rtc.yaml`; preview только для сетки |
| Админка 401 | `viewer.admin_password` в `go2rtc.yaml` |
| go2rtc просит login, viewer нет | Нормально: viewer обходит basic auth API |
| Не подхватился config | Перезапуск после сохранения или замены exe |
